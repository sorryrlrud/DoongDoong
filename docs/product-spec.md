# 둥둥 MVP 제품 규칙

## 핵심 감정

둥둥은 연결감, 해방감, 우연성, 창작의 즐거움을 고르게 담습니다.

> 내 안에서 꺼내어 버리고 싶지만 누군가 봐주었으면 좋겠다. 그렇지만 결국 사라졌으면 좋겠다.

## 불변 원칙

1. 한 병은 동시에 한 사람만 만난다.
2. 작성자와 수신자 모두 외부에는 완전히 익명이다.
3. 답장, 반응, 좋아요, 팔로우, 검색, 공유가 없다.
4. 작성자는 띄운 뒤 글·읽음·이동 경로·재표류 여부를 확인하거나 회수할 수 없다.
5. 닫힌 병에서는 지역, 언어, 날짜, 서명, 길이를 모두 숨긴다.
6. 받은 사람은 `다시 띄우기`, `버리기`, `30일 보관` 중 하나를 고른다.
7. 보관 중에도 다시 띄우거나 버릴 수 있고 30일 뒤에는 사라진다.
8. 편지 본문은 텍스트만, 10~1,000자다. 날짜와 20자 이하 개인 서명은 선택이다.
9. 하루 최대 두 병을 띄우고, 한 병을 건진 뒤 다음 병까지 12시간 기다린다.
10. 위치 권한 없이 태평양·대서양·인도양·북극해·남극해 중 하나를 고른다.
11. 국가와 언어는 별도 설정이며, 최초 작성자의 언어가 병의 불변 원문 언어가 된다.
12. 병이 수신자에게 도달하면 수신자 언어 번역을 캐시하고, 재표류 시 원문과 기존 캐시를 유지한다.

## 도착 규칙

최초 병은 1시간에서 7일 사이 무작위로 도착 후보가 됩니다. 다시 띄워질 때마다 최대 지연이 줄어듭니다.

```text
next delay = random(1 hour, max(1 hour, 7 days × 0.65 ^ drift count))
```

재표류 횟수와 이동 이력은 누구에게도 보여주지 않습니다. 같은 사용자가 자신의 서명과 날짜를 단서로 과거의 병을 우연히 다시 만날 수 있으므로, 작성자도 수신 후보에서 제외하지 않습니다.

## 상태 흐름

```text
draft → moderating → drifting → available → delivered
  delivered → drifting
  delivered → kept → drifting
  delivered → kept → deleted(expired)
  delivered → deleted
  delivered → reported
```

닫힌 병이나 열린 병을 24시간 처리하지 않으면 운영판에서는 자동으로 다시 띄우는 것을 기본값으로 합니다.
`deleted`와 `reported` 메시지는 사용자에게 다시 노출하지 않지만 운영 데이터베이스에는 남깁니다.

## 안전

- 개인정보·연락처·외부 계정·URL
- 성적 내용, 혐오, 괴롭힘, 노골적 폭력
- 자해·범죄의 구체적 실행 내용
- 광고, 구걸, 스팸, 외부 만남 유도

위 범주는 발송 전에 차단합니다. 신고된 병은 즉시 유통에서 격리하고, 제재를 위해 작성자 UID와 최소 감사 기록은 사용자에게 보이지 않는 운영 메타데이터로 보관합니다.
